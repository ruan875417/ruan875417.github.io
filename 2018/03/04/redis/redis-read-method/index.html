Redis源码剖析--Redis源码结构

发表于 2018-03-02 | 分类于 Redis | 阅读次数

由于工作中用到redis，并且redis的源码相对其他开源软件比较简单，因此先从学习redis源码开始，慢慢累积自己的技术知识。

Redis源码解析

上图中源码文件很多，必须找到一种合理的顺序来阅读各个文件。
网上找到的源码的阅读法。摘自redisredis源码解析
自底向上：从耦合关系最小的模块开始读，然后逐渐过度到关系紧密的模块。就好像写程序的测试一样，先从单元测试开始，然后才到功能测试。我在刚开始读 Redis 源码的时候，使用的就是这种方法：先从单独的数据结构模块开始，然后再过渡到高层的功能模块。
从功能入手：通过文件名（模块名）和函数名，快速定位到一个功能的具体实现，然后追踪整个实现的运作流程，从而了解该功能的实现方式。我在读阻塞列表、数据库这种流程和功能都比较复杂，和其他文件耦合也比较多的模块时，使用的就是这样的方法。
自顶向下：从程序的 main() 函数，或者某个特别大的调用者函数为入口，以深度优先或者广度优先的方式阅读它的源码。我在阅读 redis.c/serverCron() 、 redis.c/main() 和 ae.c/aeMain() 这种有明显调用者性质的函数时，使用的就是这样的方法。

通过以上方法，可以将redis源代码进行模块的拆分，由浅入深地阅读。
3.png
根据上图的模块划分，确定源码的学习路线：

第一阶段

阅读redis源码的数据结构部分：
redis的内存分配zmalloc.h和zmalloc.c
redis的动态字符串实现sds.h和sds.c
redis的双端链表实现dict.h和dict.c
redis的的字典实现dict.h和dict.c
redis的跳跃表实现，redis.h中的zskiplist结构和zskiplistNode结构，以及t_zset.c 中所有以zsl开头的函数，比如zslCreate 、zslInsert 、zslDeleteNode 等等。
Redis的HyperLogLog实现，hyperloglog.c 中的 hllhdr 结构， 以及所有以 hll 开头的函数。

第二阶段

阅读内存编码数据结构实现：
整数集合（intset）数据结构intset.h 和 intset.c
压缩列表（zip list）数据结构ziplist.h 和 ziplist.c

第三阶段

阅读数据类型实现：
redis 的对象（类型）系统实现object.c
字符串键的实现t_string.c
列表键的实现 t_list.c
散列键的实现 t_hash.c
集合键的实现 t_set.c
有序集合键的实现 t_zset.c 中除 zsl 开头的函数之外的所有函数
HyperLogLog 键的实现 hyperloglog.c 中所有以 pf 开头的函数

第四阶段

阅读Redis数据库的实现：
redis的数据库实现 redis.h文件中的redisDb结构，以及db.c文件
redis的数据库通知功能实现 notify.c
redis的RDB持久化实现代码 rdb.c
redis的AOF持久化 aof.c
以及一些独立功能模块的实现

发布与订阅功能的实现 redis.h文件的pubsubPattern结构，以及pubsub.c文件
事务功能的实现 redis.h文件的multiState结构以及multiCmd结构，multi.c文件
SORT 命令的实现 sort.c
GETBIT 、 SETBIT 等二进制位操作命令的实现 bitops.c

第五阶段

阅读客户端和服务器的相关代码：
redis的事件处理模块 ae.c/ae_epoll.c/ae_evport.c/ae_kqueue.c/ae_select.c
redis的网路链接库 anet.c和networking.c
redis的服务器端 server.c
redis的客户端 redis-cli.c
以及一些独立功能模块的实现
lua脚本 scripting.c
慢查询 slowlog.c
监视 monitor.c

第六阶段

阅读Redis多机功能的实现
redis的复制功能 replication.c
redis sentinel sentinel.c
redis的集群 cluster.c
